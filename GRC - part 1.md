- Confidentialité : Empêcher les communications d'être observées par un tiers.
- Intégrité : Garantir que les données n'ont pas été altérés.
- Disponibilité : Les données sont accessibles par les utilisateurs autorisés lorsque nécessaire.

#Asset Critique
- Activité : perte de disponibilité -> baisse de productivité
- Données Sensibles : gère des données sensibles (personnel/médical/secret)
- Interdépendance : dépendant/conditionne des tiers
- Remplacement : coût en cas de dommage/pertes
- Rareté : facilité de remplacement (identique/alternative)
- Réputation : actif compromis -> perte de réputation
- Exigences Légales : obligations légales à la protection de l'actif
- Sécurité : risques pour la sécurité global en cas de compromission
- Exposition : Visibilité sur le réseau (interne/externe et physique/virtuelle)

##Limitations

- Complexité : gros SI -> beaucoup de données -> dure à faire

#Vulnérabilités
Faiblesses exploitables affectant la confidentialité, l'intégrité, la disponibilité

- Logicielle : Bugs ou défauts dans le code qui peuvent être exploités pour compromettre un système.
- Physique : Manque de mesures de sécurité pour protéger les équipements physiques.
- Configuration : Configurations incorrectes ou insuffisantes qui laissent des failles de sécurité.
- Réseaux : Problèmes dans la conception ou la mise en œuvre du réseau qui permettent des attaques.
- MaJ : Systèmes non mis à jour avec les derniers correctifs de sécurité.

##EPSS
Modèle prédictif développé par FIRST (Forum of Incident Response and Security Teams) pour estimer la probabilité qu'une vulnérabilité soit exploitée dans la nature.

##Mécanisme
- Disponibilité des Exploits : La présence d'un exploit public augmente significativement la probabilité d'exploitation.
- Caractéristiques de la Vulnérabilité : Des attributs tels que le type de vulnérabilité et la complexité de l'exploit influencent la probabilité.
- Prédictif et Proactif : Permet aux organisations de prendre des mesures préventives avant qu'une exploitation ne se produise.
- Basé sur des Données : Utilise une vaste base de données historiques pour fournir des estimations fiables.
- Basé sur un groupe ouvert (FIRST)

##Limitations
- Dépendance aux Données Historiques : Les prédictions peuvent ne pas être précises pour les nouvelles tendances d'exploitation non encore observées.
- Nécessité de Mises à Jour Constantes : Le modèle doit être régulièrement mis à jour avec de nouvelles données pour rester pertinent.
